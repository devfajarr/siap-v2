# Release Notes - Fitur Pengajuan PKL Berkelompok (Dashboard V2)

**Versi:** 2.1.0  
**Tanggal Rilis:** 18 Mei 2026  
**Kategori:** Fitur Baru & Peningkatan Sistem (Feature & Enhancement)

---

## Ringkasan Eksekutif
Dalam rangka memfasilitasi kebutuhan praktis pelaksanaan PKL dan Tugas Akhir (TA) mahasiswa yang umumnya dilakukan secara berkelompok di satu instansi mitra yang sama, sistem Siap Polsa V2 telah ditingkatkan dengan fitur **Pengajuan Surat Permohonan Berkelompok**.

Fitur ini memungkinkan seorang mahasiswa (sebagai Ketua Pengusul) untuk memilih rekan satu timnya secara langsung dalam satu formulir pengajuan. Sistem secara otomatis mendistribusikan riwayat permohonan ke dashboard masing-masing anggota tim, dengan pembatasan hak akses ketat (otorisasi 403) yang melarang anggota untuk mengedit atau menghapus pengajuan milik ketua. Template cetak surat di sisi admin juga telah diperbarui untuk merender tabel kelompok secara dinamis tanpa merusak format surat perseorangan yang sudah ada.

---

## Spesifikasi Perubahan Teknis (Technical Changelog)

### 1. Database Schema & Models
- **Migration**: Menambahkan kolom `anggota_tim` bertipe `JSON` yang bersifat nullable pada tabel `permohonan_surats` via file migrasi `2026_05_18_214726_add_anggota_tim_to_permohonan_surats_table.php`.
- **Eloquent Model (`PermohonanSurat.php`)**: Menerapkan casting atribut otomatis `$casts = ['anggota_tim' => 'array']` untuk mempermudah manipulasi data berbasis JSON di tingkat aplikasi.

### 2. Backend Controller Mahasiswa (`V2\Mahasiswa\PermohonanSuratController.php`)
- **Penyempurnaan Query Riwayat**: Mengubah kueri pada metode `index` dengan `whereJsonContains` untuk memastikan surat permohonan muncul di halaman riwayat baik bagi mahasiswa pengusul (`mahasiswa_id`) maupun seluruh mahasiswa yang ID-nya terdaftar di `anggota_tim`.
- **Integrasi Daftar Mahasiswa Aktif (Filter Prodi & Syarat Minimal Semester)**: Mengirimkan koleksi data `daftarMahasiswa` (berisi `id`, `nim`, dan `nama_lengkap`) ke view Inertia. Daftar ini telah difilter secara spesifik agar hanya memuat mahasiswa aktif pada **Program Studi (`id_prodi`) yang sama** dengan pengusul, serta memenuhi syarat **minimal semester pelaksanaan PKL** (minimal semester 4 untuk jenjang D3, dan minimal semester 6 untuk jenjang S1/D4). Filter ini dirancang tanpa membatasi tahun masuk untuk mengakomodasi mahasiswa yang mengulang atau lintas angkatan, sekaligus mencegah salah pilih adik tingkat yang belum memenuhi syarat.
- **Sistem Otorisasi (Gate / 403 Forbidden)**: Menambahkan lapisan validasi pada metode `update` dan `destroy`. Jika `mahasiswa_id` dari permohonan tidak cocok dengan `auth()->id()`, sistem akan menggugurkan proses dan merender halaman `403 Forbidden`, mencegah anggota kelompok memanipulasi data tanpa izin.

### 3. Frontend Vue 3 / Inertia (`Mahasiswa/PermohonanSurat/Index.vue`)
- **Komponen Combobox Anggota Tim (Shadcn-Vue)**: Mengimplementasikan popover dan command input interaktif untuk pencarian dan penambahan anggota tim. Terdapat batasan maksimum 5 anggota per pengajuan.
- **Reaktivitas `useForm`**: Memperbarui skema form Inertia untuk menyertakan `anggota_tim: []`.
- **Indikator Keanggotaan (Badges)**: Menambahkan badge visual berlabel "Ketua" (warna ungu/primary) atau "Anggota" (warna amber) di samping nama jenis permohonan.
- **Modal Rincian Tim (`Dialog`)**: Menambahkan dialog modal khusus untuk melihat daftar lengkap Ketua dan Anggota (NIM, Nama Lengkap, dan Inisial).
- **Restriksi UI Berbasis Peran**: Elemen antarmuka seperti tombol "Edit" dan "Batal" disembunyikan secara otomatis apabila item surat permohonan di tabel riwayat bertipe "Anggota".

### 4. Backend Controller Admin & Template Blade
- **Eager Loading Relasi Anggota (`V2\Admin\PermohonanSuratController.php`)**: Metode `cetakDokumen` kini memeriksa eksistensi `anggota_tim`. Jika ada, sistem akan memuat data model `Mahasiswa` beserta relasi `kelas.prodi` dan `kelas.semester` menggunakan kueri `whereIn` untuk mencegah masalah N+1 Query.
- **Pembaruan Template Cetak (`ijin_pkl.blade.php` & `ijin_memperoleh_data.blade.php`)**:
  - Ditambahkan blok kondisional `@if(!empty($anggotaTim) && count($anggotaTim) > 0)`.
  - Jika permohonan bersifat kelompok, halaman cetak surat akan menampilkan tabel daftar mahasiswa yang rapi (No, Nama Lengkap, NIM, Program Studi).
  - Jika permohonan bersifat individu, halaman cetak surat tetap menggunakan tabel informasi baris konvensional (`info-table`).

## Verifikasi & Keandalan (Quality Assurance)
1. **Pengujian Fungsional**: Pengajuan baru, penambahan anggota tim, dan validasi jumlah maksimum anggota berjalan lancar.
2. **Pengujian Keamanan**: Percobaan pengubahan dan penghapusan data oleh akun berstatus anggota berhasil diblokir dengan status `403 Forbidden`.
3. **Pengujian Layout Cetak**: Format kop surat, penempatan alamat mitra, isi surat, dan tanda tangan Direktur tetap pada posisinya yang sesuai, baik saat mencetak surat tunggal maupun kelompok.
4. **Kompilasi Aset**: Aset JavaScript dan CSS telah dikompilasi ulang dengan sukses menggunakan Vite (`npm run build`).
